## 8. Riscos principais

- Modal: `SignatureModal.CANCELED` tem comentario dizendo abortar save, mas `saveRegistro` atual salva sem assinatura e mostra info; qualquer extracao deve preservar o comportamento real, nao o comentario.
- Data URL: ha dois validadores (`isSafeSignatureCaptureDataUrl` e `isSafeRegistroSignatureSrc`) com regex equivalente; risco de divergir.
- Storage/upload: `saveSignatureForRecord` escreve cache local antes do upload e retorna `null` em fallback; payload usa boolean `true` nesse caso.
- Fila offline/fallback: `cooltrack-sig-pending-upload` e flush atualizam `registros.assinatura` depois; mudar shape quebra PDF/historico.
- Gating de plano: Free pula modal no save e mostra upsell no hint; Plus+ ve modal automatico no save.
- Integracao com payload/saveRegistro: assinatura fica entre captura de fotos pendentes e upload de fotos CP-F.
- Integracao com fotos/evidencias: ordem relativa foi preservada no CP-F e precisa continuar intacta.
- Integracao com PDF/relatorio: domain/pdf importa UI signature para resolver dados; risco arquitetural existente.
- Integracao com historico: viewer usa resolver async e fallback local; dados de outro dispositivo podem aparecer indisponiveis.
- Selectors/data-actions/classes: roots/actions/classes sao publicos e protegidos por CP-B.
- Import circular: mover save signature para feature pode puxar UI component/signature ou core/state de volta.
- Regressao silenciosa: root ausente, bridge ausente, modulo dinamico ausente e storage indisponivel tendem a falhar sem bloquear save.
